## Critical (blocks daily use or looks broken)

### 1. Composer: Send button + text clipped behind home indicator
**Screen**: Chat
**Status**: FIX APPLIED this session
- SafeAreaView only had `edges={['top']}` — bottom was unprotected
- Added `useSafeAreaInsets().bottom` as paddingBottom on MessageComposer root when keyboard is hidden
- **Verify**: Type a message without triggering software keyboard — send button should be fully visible above home indicator

### 2. "Starter plan" and "Unlock Auto-Pilot — available on paid plans" shown to founder
**Screen**: Settings > AI Usage / Auto-Pilot
**Root cause**: `usageStats?.tierLabel || 'Starter'` defaults to "Starter" in personal mode. `hasPaidAutoPilot` requires `normalizedTier === 'professional'` which personal mode never sets.
**Fix**: In personal mode, the founder should see either:
- No plan label at all (hide commercial tier UI), OR
- "Personal" / "Founder" label instead of "Starter"
- Auto-Pilot should be unlocked for the founder regardless of tier
**File**: `src/components/SettingsScreen.tsx` lines 177-182

### 3. Conversation list items have zero left padding — avatars/text touch screen edge
**Screen**: Inbox
- Avatar circles for DD, HP, AD are flush against the left edge with no margin
- Guest name, message preview, date range all start at x=0
- Needs `paddingHorizontal: 16` on conversation rows
**File**: Likely `src/components/ConversationItem.tsx`

### 4. ~~Yellow debug rectangles~~ — NOT A BUG
React Native offscreen rendering debug overlay. Disable in dev menu before App Store screenshots.

## High (noticeable quality issues)

### 5. Draft card color scheme feels wrong — yellow-green doesn't read as "AI"
**Screen**: Chat (AI Draft Ready, Editing AI Draft)
- The draft card uses a yellow-green tint that blends with the teal/green app theme
- "Send Draft" button is teal-on-yellow — low contrast, hard to distinguish from other buttons
- **Suggestion**: Use a distinct but complementary color for AI content — light purple/lavender or soft blue — to clearly separate "AI generated" from "host written"
- The orange host message bubbles are strong and clear; AI content should have equal visual weight but different identity

### 6. AI Draft confidence (57%) doesn't affect visual treatment
**Screen**: Chat (AI Draft Ready showing "57% · Positive")
- A 57% confidence draft looks identical to a 95% confidence draft
- Low confidence should feel less certain: muted colors, smaller "Send Draft" button, or a warning nudge
- High confidence should feel confident: bold presentation, prominent send
- **This is a product differentiator** — competitors don't show confidence at all

### 7. "Editing AI Draft" state has dangerous button placement
**Screen**: Chat (Editing AI Draft)
- "Save & Send" (destructive/irreversible) is directly next to "Clear" (also destructive)
- No confirmation before sending edited draft
- "Done" label above the text area is ambiguous — does it save? dismiss? send?
- **Fix**: "Done" should be "Save" or removed; "Save & Send" needs either spacing or a confirm step

### 8. Today's Briefing card takes up too much vertical space
**Screen**: Inbox
- The briefing card with 2 items + "+1 more" takes ~40% of visible screen
- On smaller phones this pushes the actual conversation list below the fold
- **Fix**: Make it collapsible (already has chevron) but default to collapsed after first view of the day, or make it more compact

### 9. Filter tabs cut off on the right edge
**Screen**: Inbox
- "Check-i..." is truncated — user can't see if it says "Check-in" or "Check-out"
- The horizontal scroll is subtle — no visual indicator there are more tabs
- **Fix**: Either make tabs smaller to fit, add a fade/gradient at the right edge, or show a scroll indicator

### 10. Message preview in Inbox shows "Dean:" prefix for guest messages
**Screen**: Inbox
- "Dean: Thanks for looking into it" — the guest name is redundant since it's already the row header
- Host messages show "You:" which is correct
- **Fix**: Remove guest name prefix from preview, keep "You:" prefix for host messages

## Medium (polish for App Store quality)

### 11. AI Learning screen has too many sections with no visual hierarchy
**Screen**: AI Learning
- Learning Status, Coverage, Top Questions, Accuracy Trend, What I Learned, Style Preferences, Voice DNA, Historical Importer, Background Sync, Reset, How It Works — all in one long scroll
- No clear grouping or progressive disclosure
- **Fix**: Group into 2-3 tabs or collapsible sections: "Voice Profile" (DNA, style, what I learned), "Training Data" (importer, coverage, accuracy), "System" (sync, reset, how it works)

### 12. Accuracy Trend shows "Need 9 more drafts" with no visual progress
**Screen**: AI Learning > Accuracy Trend
- "Approval Rate: Need 9 more drafts" — just text, no visual indicator of progress toward the threshold
- "Usable Rate: —" and "Total: 1" with "100% This Week" — misleading with only 1 data point
- **Fix**: Show a progress ring or bar toward the 10-draft threshold. Don't show "100%" stats on 1 draft.

### 13. Style Preferences sliders have no labels for the range
**Screen**: AI Learning > Style Preferences
- Formality and Warmth sliders exist but the screenshot shows no min/max labels
- User can't tell what direction means what
- **Fix**: Label endpoints: "Casual ←→ Formal", "Cool ←→ Warm"

### 14. Property Knowledge at 88% — what's missing isn't obvious
**Screen**: Property Knowledge
- "88% configured" with a progress ring, but which sections are incomplete?
- User has to scroll through all sections to find gaps
- **Fix**: Highlight incomplete sections at the top, or show a "Complete these to reach 100%" call-to-action

### 15. "Copy Voice DNA" feature purpose unclear
**Screen**: AI Learning > Voice DNA
- "Your portable communication fingerprint. Copied DNA can be pasted into any AI to clone your voice."
- This is actually a cool feature but it's buried and the copy target isn't clear
- **Fix**: Make this more prominent — it's a unique selling point. Show a preview of what gets copied.

### 16. Chat header property banner is always expanded
**Screen**: Chat
- The teal "2 Hot Tubs+Pool+MiniGolf+Sl... 12 guests · Mar 7 - Mar 13" banner takes a lot of vertical space
- Plus the "Inquiry" badge bar below it
- Together they take ~120pt before messages start
- **Fix**: Collapse to a single line after scrolling up, or make it tappable-to-expand

### 17. "Show reasoning & limits" / "Hide reasoning & limits" toggle
**Screen**: Chat (AI Draft Ready)
- Good feature, but the text is small and the sparkle icon is subtle
- When expanded, "How I Arrived at This" with "AI-generated response (57% confidence)" is useful
- **Fix**: Consider making the confidence badge the primary toggle (tap 57% to see reasoning)

### 18. No empty state for Check-out filter
**Screen**: Inbox (Check-out tab)
- When filtering to Check-out (1), only 1 conversation shows with lots of blank space below
- No illustration or helpful text for the empty area
- **Fix**: Add contextual empty state: "All check-outs handled" or show upcoming check-outs

## Low (polish / nice-to-have)

### 19. Tab bar icons could use more visual weight
**Screen**: All screens (bottom tab bar)
- Inbox, Calendar, Settings icons are thin line-style (lucide)
- The active teal color is subtle against the white background
- Consider filled variants for active state (e.g., filled message square for active Inbox)

### 20. Unread badge on Inbox tab uses primary teal — could use red for urgency
**Screen**: Tab bar
- Badge showing "3" is teal — same as active tab color, doesn't pop
- Industry standard is red/orange for unread badges
- Minor, but red draws the eye faster

### 21. Search bar placeholder text could be more helpful
**Screen**: Inbox
- "Search guests, properties..." is fine but could be more action-oriented
- "Search conversations..." or "Find a guest or property..." reads better

### 22. "Disconnect Hostaway" at bottom of Settings is dangerously easy to hit
**Screen**: Settings (bottom)
- Destructive action with no visual separation from regular items
- **Fix**: Add red text, extra spacing, or move to a sub-menu

### 23. Version number "1.0.0" in Settings should show build number
**Screen**: Settings > About
- "Version 1.0.0" — should include build number for debugging: "1.0.0 (42)"

### 24. Background Sync section could show last sync time
**Screen**: AI Learning > Background Sync
- "Continue in Background — Sync history even when the app is closed"
- No indication of when it last ran or if it's currently syncing
- **Fix**: Add "Last sync: 2 minutes ago" or a subtle status indicator

## Feature Gaps (competitors do this, we should too)

### 25. No message read receipts or delivery status in inbox list
- Conversation previews don't show if the guest has read the last message
- Airbnb shows read receipts — hosts care about this

### 26. No quick-reply from inbox (must open full chat)
- Competitors let you swipe to quick-reply or type from the inbox row
- Current flow: tap conversation → wait for chat load → scroll to bottom → type

### 27. No notification badge differentiation
- All 3 unread conversations show the same priority
- A refund risk (Dean Decker) should look different from a simple inquiry

### 28. No guest sentiment indicator in inbox
- The AI already computes sentiment ("Positive" badge in draft card)
- Surface this in the conversation list: guests who are unhappy need attention first

### 29. Calendar tab has no visible connection to inbox
- Checking in/out guests from calendar should link to their conversation
- Currently these feel like separate features
